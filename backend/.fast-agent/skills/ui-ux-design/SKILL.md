# UI/UX Design (Designer)

> **⚠️ RESERVED:** This skill is prepared for the future Designer role. Not currently in use.

## Purpose

Guide the Designer role in:
- Creating user-centered interfaces
- Applying visual design principles
- Ensuring accessibility (WCAG compliance)
- Maintaining design consistency

## Design Process

### 1. User Research
- Understand target users and their goals
- Define user personas
- Map user journeys and pain points

### 2. Information Architecture
- Organize content and features logically
- Create navigation structure
- Define page hierarchy

### 3. Wireframing
- Low-fidelity layouts for structure
- Focus on content placement and flow
- Get feedback before visual design

### 4. Visual Design
- Color palette — harmonious, accessible contrast ratios
- Typography — readable, consistent hierarchy
- Spacing — consistent rhythm and whitespace
- Icons and imagery — purposeful, not decorative

### 5. Interaction Design
- Hover states, transitions, micro-animations
- Loading states and error states
- Responsive behavior across breakpoints

## Accessibility Checklist

- Sufficient color contrast (WCAG AA: 4.5:1 for text)
- Keyboard navigation support
- Screen reader compatibility (semantic HTML, ARIA labels)
- Focus indicators visible
- Alt text for images

## Design Principles

- **Clarity over cleverness** — Users should immediately understand the interface
- **Consistency** — Same patterns for same actions everywhere
- **Feedback** — Every action should have a visible response
- **Progressive disclosure** — Show essentials first, details on demand
- **Accessibility first** — Design for all users from the start
