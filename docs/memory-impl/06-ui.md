## 1. SSE events — `frontend/src/stores/agents.js`

`processEvent(event)` switch (~line 158) handles ONLY these four; all other
memory events exist for the audit trail / Memory page fetches:

- `memory_candidate_created` → increment per-agent pending-candidate badge
  (forward to a new `useMemoryStore`, mirroring how approval events forward
  to `useApprovalsStore` at ~line 330).
- `memory_candidate_approved` / `memory_candidate_rejected` → update badge +
  candidate list if loaded.
- `memory_indexed` → refresh index-status if the Memory tab is open.
- `retrieval_degraded` → set a transient degraded flag (shown as a small
  warning chip in chat + Memory tab; auto-clears on next healthy
  retrieval).

Store seeding rule (CLAUDE.md anti-pattern): any AgentCard/tab gate reading
`useMemoryStore` must be seeded in the parent view's `onMounted` — no
cross-store gates rendering stale on a fresh tab.

## 2. Chat evidence drawer — `ChatMessages.vue`

Follow the existing tool-details accordion pattern (`expandedTools`,
~lines 263-282). No new message bubbles, no modal:

- Assistant messages whose turn used evidence get a compact chip under the
  bubble: `🧠 3 memories used` / `🧠 dùng 3 ký ức`. Data arrives with the
  message metadata (evidence ids + excerpts from the retrieval result).
- Click → in-place expansion listing per-evidence: excerpt, source label
  (type + humanized timestamp), authority badge, confidence. Each row has
  three feedback buttons (useful / irrelevant / incorrect) → POST to the
  feedback endpoint; optimistic UI (button state flips immediately,
  reverts + toast on failure).
- During deep (Level 2) retrieval only, show lightweight status text in the
  progress area (existing thinking/status mechanism): "Searching previous
  decisions" / "Đang tìm các quyết định trước". Level 1 must be visually
  silent.
- Mobile: the expansion renders as a bottom sheet instead of inline
  accordion.

## 3. Agent Memory tab — `frontend/src/views/AgentDetail.vue`

Add `memory` to `validTabs` (~line 23), placed next to `context`;
lazy-load on activation like the `context`/`versions` tabs (watch pattern
~line 185). Tab content = new `AgentMemoryPanel.vue` with sections:

1. **Overview**: counts per memory type, pinned token budget bar
   (used/total), index status (incl. degraded/dead-letter indicators with a
   re-queue action).
2. **Memories list**: filter by type/status/scope + text search (calls
   `POST /api/agents/{name}/memory-search`); row → detail with content,
   provenance (sources + excerpts), authority, confidence, validity,
   version history. Actions: edit (creates new version), pin/unpin,
   archive, delete, rollback. Delete/rollback are confirmable; archive is
   the default destructive action (reversible) — copy explains that.
3. **Candidates**: pending queue with approve/reject (badge fed by SSE);
   shows why a candidate needs approval (conflict, sensitivity, ...).
4. **Proposals**: procedural/Skill/MCP proposals awaiting approval.
5. **Retrieval runs**: recent `retrieval_runs` with level, timings, tokens,
   cache-hit — the debugging surface for "why did/didn't it recall".

Mobile: tables collapse to cards; actions in an overflow menu.

## 5. UX quality bar (CLAUDE.md item 11 — applies to every element above)

- Every action visibly responds <200ms (button busy state / optimistic
  update); loading = skeleton or pulse, never blank.
- Errors are actionable: what failed + what to do ("Qdrant unreachable —
  check connection in Settings → Memory" / vi equivalent).
- Destructive actions confirmable; archive-first philosophy.
- Match existing tokens (spacing, color, typography) — reuse the components
  the approvals/compaction UIs use; no new design language.
